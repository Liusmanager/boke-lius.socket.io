React Router v5 vs v6 核心区别

React Router从v5升级到v6时进行了架构重构，主要区别集中在路由配置、嵌套路由、导航机制等方面。以下是关键差异点：

路由配置方式

v5采用<Switch>组件进行路由匹配，按顺序匹配第一个符合条件的路由

v6改用<Routes>组件，所有路由必须包裹其中，且路径匹配更智能

路由渲染方式

v5使用component或render属性定义渲染内容

v6统一使用element属性，直接传递JSX元素

嵌套路由实现

v5需在父路由组件中手动插入子路由

v6采用更直观的嵌套结构，父路由需添加/*后缀

编程式导航

v5使用history对象进行操作

v6改用useNavigate钩子

路由参数获取

v5通过match.params获取

v6使用useParams钩子

重定向处理

v5使用<Redirect>组件

v6改用<Navigate>组件

路由优先级

v5按Route定义顺序匹配，需手动添加exact属性

v6自动选择最匹配的路由，无需exact

相对路径支持

v6新增相对路径概念，嵌套路由可以基于父路径自动计算完整路径，而v5必须写完整路径。

钩子API变化

v6引入新钩子：useRoutes、useNavigate、useOutlet等，同时移除v5的withRouter高阶组件。

这些改变使得v6的路由配置更简洁、更符合现代React开发模式，但需要开发者适应新的API设计理念。升级时需特别注意嵌套路由结构和导航方式的调整。
